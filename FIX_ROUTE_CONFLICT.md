# 🚨 СРОЧНОЕ ИСПРАВЛЕНИЕ КОНФЛИКТА МАРШРУТОВ

## Проблема
Конфликт маршрутов Next.js:
- `app/invite/[token]/page.tsx` ❌ (конфликтует)
- `app/invite/[workspaceId]/[token]/page.tsx` ✅ (правильный)

## Решение на сервере

Выполните эти команды на сервере:

```bash
cd /var/www/SaaS-polatform

# 1. Удалить конфликтующий файл
rm -rf app/invite/\[token\]

# 2. Проверить что остался только правильный маршрут
ls -R app/invite/

# Должно показать:
# app/invite/
# └── [workspaceId]/
#     └── [token]/
#         └── page.tsx

# 3. Очистить кэш Next.js
rm -rf .next

# 4. Пересобрать проект
npm run build

# 5. Перезапустить
pm2 restart saas-platform
```

## Почему это важно

Next.js не может определить, что такое `/invite/something`:
- Это `token` или `workspaceId`?
- Из-за этого 404 ошибки при принятии приглашений

## Результат после исправления

- ✅ Email ссылки работают: `/invite/TOKEN`
- ✅ Уведомления работают: кнопка "Просмотреть" редиректит в workspace
- ✅ Нет 404 ошибок в консоли
